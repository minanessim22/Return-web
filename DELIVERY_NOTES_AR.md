# ملاحظات التسليم - مشروع RETURN

هذه النسخة مبنية على **الفرونت الأصلي** الذي تم رفعه، مع تقوية الباك إند وربط الصفحات الأصلية به بدون حذف شاشات الفرونت.

## ما تم تنفيذه

### 1) تقوية الباك إند
- بناء طبقة API كاملة داخل Next.js App Router
- إدارة جلسات آمنة عبر Cookies مع تخزين Hash للجلسة فقط
- تشفير كلمات المرور باستخدام `scrypt`
- ترحيل كلمات المرور القديمة تلقائيًا عند تسجيل الدخول
- قفل مؤقت للحساب بعد محاولات فاشلة متكررة
- Rate limiting للـ login والتسجيل وإعادة تعيين كلمة المرور
- تدقيق أمني عبر Audit Logs
- تخزين JSON منظم وقابل للعرض السريع أثناء التسليم والديمو

### 2) التحقق بالبريد الإلكتروني
- عند إنشاء الحساب يتم إصدار كود تحقق مكوّن من 6 أرقام
- في حالة عدم وجود خدمة بريد خارجي، يتم حفظ الرسائل في:
  - `src/data/outbox.json`
- وفي وضع التطوير يظهر كود التحقق/إعادة التعيين أيضًا داخل شاشة المصادقة لتسهيل الديمو
- في حالة توفير `RESEND_API_KEY` و `EMAIL_FROM` يتم إرسال البريد فعليًا

### 3) ربط الشاشات الأصلية بالباك إند
- شاشة Login / Sign up الأصلية
- شاشة Report Missing / Found الأصلية
- شاشة Case Details الأصلية
- شاشة Missing الأصلية أصبحت تعرض الحالات النشطة لكل الحسابات
- Lost Dashboard و Found Dashboard أصبحا يقرآن بيانات حقيقية بدل الكروت الوهمية
- التنقل بالروابط `?tab=` أصبح يعمل بين الشاشات الأصلية والداشبورد
- مسارات `/devices` و `/profile` أصبحت تعيد التوجيه إلى التبويب الصحيح بدل صفحات 404
- إنشاء QR وربط NFC
- مسح QR وفتح Identification Profile

### 4) التحسينات الاحترافية الجديدة
- **تفاصيل الماتشات**: زر `1 match / 2 matches` أصبح يفتح صفحة الحالة مع قائمة كل العناصر المطابقة
- **التواصل بين الطرفين**: كل ماتش يعرض بيانات الطرف الآخر (الاسم / اليوزرنيم / الإيميل / الهاتف) حسب صلاحية الوصول
- **شات حقيقي**: تم تحويل شاشة الشات من واجهة شكلية إلى محادثات حقيقية مرتبطة بالماتشات
- **تأكيد/رفض الماتش**: يمكن تأكيد الماتش النهائي أو رفض الماتشات الخاطئة من صفحة الحالة
- **تبويب Matches منفصل**: عند تأكيد الماتش النهائي تنتقل الحالات من Missing / Found إلى تبويب Matches بدل بقائها ضمن الحالات النشطة
- **تحسين منطق الماتش**: تم تشديد قواعد المقارنة ورفع العتبة لتقليل الماتشات الخاطئة
- **البروفايل الحقيقي**: تبويب Profile أصبح يقرأ بيانات الحساب الحقيقي من `/api/me` ويتيح تعديلها وحفظها
- **تعديل التقرير بالكامل**: من صفحة `case-details` أصبح يمكن تعديل كل بيانات البلاغ تقريبًا (الاسم، النوع، العمر، الوصف، الموقع، الوقت، الحالة، الهاتف...)
- **فلترة احترافية**: صفحات Missing / Found / My Reports أصبحت تدعم البحث والفلترة حسب التصنيف والحالة ووجود ماتشات

## تشغيل المشروع
1. انسخ `.env.example` إلى `.env.local`
2. شغّل:
```bash
npm install
npm run dev
```
3. افتح:
- `/login`
- `/guest-homepage`

## مسارات مهمة أثناء الديمو
- `/lost-dashboard`
- `/found-dashboard`
- `/missing`
- `/chat`
- `/case-details?caseId=<CASE_ID>&view=matches`

## ملاحظات مهمة
- تم الإبقاء على بعض مفاتيح `localStorage` لأغراض التوافق مع الشاشات الأصلية، لكن مصدر البيانات الأساسي أصبح الباك إند.
- فحص TypeScript يمر بنجاح:
```bash
tsc --noEmit
```
- إذا فشل `next build` على لينكس بعد نقل المشروع، احذف `node_modules` ثم أعد التثبيت لأن النسخة المرفوعة أصلًا احتوت على ملفات SWC خاصة بويندوز.
